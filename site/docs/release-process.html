Release process

Tagged releases that stay predictable.

The release workflow verifies the project, builds multi-platform archives, writes checksums, and publishes GitHub release notes from a reviewed tag.

Back to site Open source doc

Before tagging

Ensure CI is green, update the changelog, and run the demo scripts locally so the product examples still match the shipped behavior.

Release outputs

Each release should publish platform archives, a checksum file, and generated GitHub release notes so users have a clear install and verification path.
