Migrate Deliberately

Apache NiFi flow upgrade tool for safer migrations.

nifi-flow-upgrade-advisor is a terminal-first Apache NiFi upgrade tool for flow upgrade reviews, deterministic rewrites, and target validation. It helps platform teams and flow owners answer one practical question: is this flow safe to move to the target NiFi version?

Why it feels easier to use

You get an answer, not just an export file.

The best upgrade tool is not the one that makes the biggest promise. It is the one that tells you whether the Apache NiFi upgrade path is viable, what changed, what can be rewritten safely, and where you still need a person in the loop.

For platform teams

Gate risky upgrades before they reach production

JSON and Markdown reports make it easy to add policy checks in CI and still give human reviewers a readable migration summary.

For flow owners

See the exact processor or service that triggered the finding

Findings carry component name, type, scope, path, and evidence so people do not have to reverse-engineer the problem from raw exports.

For delivery teams

Publish into the handoff that already fits your workflow

Use filesystem outputs, Git-backed registry trees, or NiFi Registry version import without turning migration logic into hidden runtime behavior.

Workflow

A simple path from “can we upgrade?” to “here is the reviewed result.”

01

Analyze the source artifact

Load flow.json.gz, versioned snapshots, NiFi Registry exports, Git-backed registry directories, or legacy flow.xml.gz and compare them against the selected target version.

02

Rewrite only what is mechanically safe

Property renames, type replacements, and other deterministic transforms are applied only when the rule pack makes the mapping explicit and confidence is high.

03

Validate against the target

Check target NiFi version, extension inventory, and process-group readiness before you attempt an import or update.

04

Publish deliberately

Push the reviewed artifact into a filesystem handoff, a Git-backed registry layout, or NiFi Registry without hiding what changed.

Safer than export and import

Secrets stay external, findings stay explainable.

NiFi intentionally does not export sensitive values in a way this tool can recover. The advisor preserves references and structure, highlights what needs rehydration, and avoids pretending it can recreate hidden values.

  • Preserves exported structure and non-sensitive configuration
  • Flags deprecated and removed components before import
  • Keeps manual architectural decisions visible instead of guessing
nifi-flow-upgrade run \
  --source ./flow.json.gz \
  --source-format flow-json-gz \
  --source-version 1.27.0 \
  --target-version 2.0.0 \
  --rule-pack ./examples/rulepacks/nifi-1.27-to-2.0.official.yaml \
  --publish \
  --publisher fs \
  --destination ./published \
  --output-dir ./out
